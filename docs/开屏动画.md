# MAXPLUS 独立站首页开屏动画需求说明（Three.js 实现）

## 一、项目目标

为品牌 **MAXPLUS** 的独立站首页制作一个具有强烈视觉冲击力的开屏动画。
 动画核心表现为：**一个排球从远处高速飞向镜头，撞击屏幕后产生冲击反馈，再向后弹回，最后在画面中心展示品牌名称 MAXPLUS。**

整体目标：

- 强化用户进入网站第一秒的品牌记忆点
- 体现排球运动的力量感、速度感、爆发感
- 提升品牌专业度与高级感
- 动画结束后自然衔接到首页首屏内容

------

## 二、技术实现要求

### 技术方案

使用 **Three.js** 作为核心渲染方案完成该开屏动画。

### 推荐技术栈

- **Three.js**
- 原生 JavaScript / TypeScript 均可
- GSAP 可作为时间轴与动画补间辅助工具
- 如有必要，可使用 EffectComposer 做后期特效处理

### 实现原则

1. 动画需要具备 **3D 纵深感**
2. 排球飞行过程必须体现真实的远近变化、速度变化和旋转感
3. 撞击屏幕时要有明显的“击中镜头”视觉反馈
4. 动画总时长控制在 **2.5 秒到 3.5 秒**
5. 不可过长，避免影响用户浏览
6. 动画结束后要平滑切换到网站首页主内容，不要突兀闪断
7. 需要兼容桌面端和移动端，但桌面端为优先版本
8. 代码应尽量模块化，便于后续替换球体材质、品牌文字、背景颜色和动画节奏

------

## 三、核心创意说明

### 动画主线

用户打开网站后，首先看到一个相对简洁、有空间感的背景场景。
 此时一个排球从画面远处出现，体积较小，并带有轻微旋转。随后排球沿 Z 轴方向快速朝镜头飞来，速度逐渐提升，体积迅速变大，营造出高速逼近用户视角的压迫感。

当排球接近镜头前方时，在屏幕中心发生一次强烈“撞击屏幕”的效果。此时画面需要有冲击感，比如：

- 镜头轻微震动
- 短暂的缩放或位移反馈
- 冲击波扩散
- 光晕或粒子爆发
- 可选的玻璃震裂/空气压缩波纹效果

撞击后，排球发生真实反弹，向后退回一定距离。
 在撞击中心区域或排球弹回后的焦点位置，品牌名称 **MAXPLUS** 逐渐显现并最终稳定停留在画面中央，成为最终视觉主角。

------

## 四、视觉风格要求

### 整体风格关键词

- 运动感
- 爆发力
- 速度感
- 专业
- 高级
- 简洁有力量

### 背景风格建议

背景不需要复杂场景建模，建议采用简洁、聚焦品牌的方式，例如：

- 深色纯净背景
- 带轻微空间雾感的渐变背景
- 微弱灯光氛围，突出球体和品牌文字

目标是让排球与品牌名称成为绝对视觉中心。

### 排球表现要求

排球需要具备真实 3D 材质感：

- 有清晰的表面拼接纹理
- 有合理的高光与阴影
- 飞行时带旋转
- 靠近镜头时有明显透视放大效果
- 不要卡通化，偏真实运动品牌质感

### 品牌文字表现要求

品牌名称：**MAXPLUS**

建议表现方式：

- 不要一开始就显示
- 应在排球撞击后被“砸出来”或“冲击显现”
- 可以采用淡入 + 模糊清晰化 + 轻微位移稳定的方式
- 字体整体偏粗体、现代、运动品牌风格
- 最终文字需要清晰稳定地停留在画面中心

------

## 五、动画分镜与时序要求

以下是建议时序，前端可以在此基础上微调节奏：

### 1）开场静止阶段（0s - 0.4s）

- 页面加载完成后，先出现一个简洁的背景空间
- 排球暂未明显出现，或者仅在远处有一个很小的球体轮廓
- 氛围保持克制，为后续冲击做铺垫

### 2）排球远距离出现并启动（0.4s - 1.0s）

- 排球从远处进入用户视野
- 球体较小，位于画面中轴附近，可略有偏移
- 球开始向镜头方向飞来
- 同时带有适度旋转，增强运动真实性

### 3）高速逼近阶段（1.0s - 1.6s）

- 排球快速沿 Z 轴冲向镜头
- 体积迅速放大
- 速度感明显增强
- 可增加轻微运动拖尾、动态模糊、光晕拉伸等效果
- 镜头或 FOV 可适度变化，增强临场冲击感

### 4）撞击阶段（1.6s - 1.8s）

- 排球在屏幕中心附近发生一次撞击反馈
- 需要明显的视觉爆发点
- 表现建议至少包含以下两种：
  - 镜头震动
  - 冲击波扩散
  - 短时间高亮闪光
  - 粒子散射
  - 屏幕波纹或裂纹效果（可选）
- 撞击瞬间应是整段动画最强的视觉峰值

### 5）反弹阶段（1.8s - 2.2s）

- 排球从撞击位置向后弹回
- 弹回距离不宜太远
- 速度由快变慢，形成真实缓冲感
- 排球可停留在背景中略偏后的位置，也可逐渐淡出弱化

### 6）品牌显现阶段（2.0s - 3.0s）

- 品牌名称 **MAXPLUS** 在撞击后开始显现
- 建议从撞击点或中心区域逐步展开
- 文字先轻微模糊，再迅速清晰
- 可伴随轻微 scale 或 opacity 动画
- 最终稳定停留于中心位置

### 7）结束与首屏衔接（3.0s 以后）

- 开屏动画结束后，平滑切入首页 Hero 区域
- Three.js 场景可逐渐淡出，或转为首页背景的一部分
- 不要突然黑屏或直接跳切

## 六、Three.js 场景与对象设计要求

### 1. 场景 Scene

- 创建独立开屏动画场景
- 背景保持简洁
- 可加入少量环境雾效提升空间层次

### 2. 相机 Camera

- 使用 PerspectiveCamera
- 需要明显体现球体从远到近的纵深效果
- 可在撞击瞬间对 camera 做短促震动或轻微位移
- 也可在飞行过程中微调 FOV，增强压迫感

### 3. 灯光 Lighting

建议至少包含：

- 主光源：突出排球表面明暗关系
- 辅助光源：减少过死阴影
- 轮廓光或背光：让球体与背景分离

目标：

- 排球材质层次清晰
- 撞击时可短暂增强光效

### 4. 排球模型

实现方式二选一：

- 使用真实排球 3D 模型（优先）
- 或使用 SphereGeometry + 自定义贴图近似实现

要求：

- 有清晰贴图
- 可旋转
- 可独立控制位置、缩放、反弹、材质变化
- 靠近镜头时不能失真严重

### 5. 品牌文字

可选方案：

- 使用 Three.js TextGeometry 做 3D 文本
- 或将品牌文字作为 2D DOM 层叠加在 Canvas 上方
- 如果追求开发效率和显示清晰度，建议文字用 DOM/CSS 实现，Three.js 负责排球和特效

要求：

- 文字最终显示必须足够清晰
- 不可因为后处理导致识别度下降
- 动画结束后的品牌名称应保持稳定，不抖动

------

## 七、特效要求

### 必做特效

1. 排球飞行动画
   - 位移
   - 缩放
   - 旋转
2. 撞击反馈
   - 镜头震动
   - 冲击感爆发
3. 品牌文字显现
   - 渐显 / 清晰化 / 稳定落位

### 建议加入的增强特效

1. 运动拖尾
   - 提升高速飞行质感
2. 冲击波
   - 撞击时从中心向外扩散
3. 粒子爆发
   - 撞击瞬间少量碎片、粉尘或光粒散开
4. 屏幕波纹 / 折射
   - 体现“打到屏幕”的感觉
5. 瞬时光闪
   - 增强峰值冲击

### 可选特效

- 玻璃裂纹
- 色差
- 镜头畸变
- 屏幕短暂冻结帧效果

注意：
 这些效果只能作为增强，不能过度花哨，核心仍然是排球 + 冲击 + MAXPLUS 显现。

------

## 八、交互与播放逻辑

### 播放规则

- 首次进入首页时自动播放一次
- 播放完成后不应反复循环
- 可通过 sessionStorage / localStorage 控制同一会话仅播放一次，避免影响体验

### 用户可操作性

- 动画播放过程中不建议完全锁死页面太久
- 可允许用户点击跳过开屏
- 若用户快速滚动或操作页面，可考虑提前结束动画并直接进入首页内容

### 降级逻辑

当设备性能不足、浏览器不支持或用户开启减少动态效果偏好时，应提供降级方案：

- 简化为 2D 动画
- 或直接显示品牌 Logo 淡入，不播放完整 3D 开屏

------

## 九、性能要求

### 性能目标

- 首屏开屏动画加载尽量轻量
- 避免因 3D 资源过大导致首页打开过慢
- 动画运行保持流畅，尽量接近 60fps
- 尤其注意移动端中高端机型表现

### 优化要求

1. 排球模型面数控制合理
2. 纹理贴图大小适中，避免过大
3. 后处理特效不宜过多叠加
4. 动画结束后释放不必要资源
5. 尽量减少持续渲染压力，可在结束后停止渲染或降低渲染频率

------

## 十、响应式要求

### 桌面端

- 作为主设计版本，需重点打磨
- 排球的冲击路径和文字位置以桌面视觉为标准

### 移动端

- 保留核心创意
- 适当减少特效复杂度
- 保证品牌名称显示完整
- 避免排球过大导致超出屏幕或冲击过猛引起观感不适

## 十二、验收标准

以下条件满足则视为符合需求：

1. 用户进入首页时可看到排球从远处飞来
2. 排球具有真实的 3D 旋转、透视和速度感
3. 撞击屏幕时有明显冲击反馈
4. 撞击后排球有反弹动作
5. 品牌名称 **MAXPLUS** 在动画后清晰展示
6. 动画整体节奏紧凑，控制在 2.5 到 3.5 秒左右
7. 动画结束后能自然衔接首页内容
8. 桌面端体验流畅，移动端可正常降级或适配
9. 不因动画导致首页首屏严重卡顿或加载过慢
