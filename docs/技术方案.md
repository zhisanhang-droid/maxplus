# MaxPlus 运动品牌独立站技术方案

## 1. 技术方案目标

本方案基于现有静态首页原型，落地为一套可维护、可扩展、可上线的运动品牌独立站技术架构。

技术前提如下：

- 前端技术：Vue 3
- 后端技术：Node.js
- 当前原型视觉风格必须保留
- 站点需兼顾品牌展示、SEO、商品展示和批发询盘

## 2. 方案原则

- 保风格：严格继承当前静态稿的布局、配色、字体层级和动效节奏。
- 重 SEO：营销页和商品页必须具备搜索引擎友好能力。
- 可运营：首页内容、商品、询盘、订阅均可后台管理。
- 可扩展：初期满足品牌展示站上线，后续可扩展会员、购物车、支付、活动、分销与外部系统对接。
- 前后端分离：前端聚焦展示与交互，后端提供业务 API 与后台管理能力。

## 3. 与现有原型的映射关系

当前原型文件：

- `index-sporting.html`
- `styles-sporting.css`
- `script-sporting.js`

正式项目中建议拆分为以下 Vue 组件：

- `AppHeader`：顶部公告、Logo、导航、语言切换、CTA
- `HomeHero`：首页 Hero 轮播
- `HomeHighlights`：品牌亮点卡片
- `FeaturedProducts`：精选商品模块
- `CategoryShowcase`：分类展示模块
- `DirectionReview`：品牌说明/评价模块
- `ContactLeadForm`：联系与批发表单
- `AppFooter`：页脚
- `SubscribeWidget`：右下角订阅浮层

现有 JS 交互迁移如下：

- 语言切换：迁移为 `vue-i18n`
- 本地语言记忆：保留 `localStorage`
- 轮播逻辑：迁移为 Vue 组件内部状态管理
- 移动菜单：迁移为响应式菜单组件
- 表单演示提交：替换为真实 API 提交
- reveal 动效：保留 `IntersectionObserver` 实现

## 4. 推荐技术栈

## 4.1 前端

- `Vue 3`
- `Vite`
- `Vue Router`
- `Pinia`
- `Vue I18n`
- `TypeScript`
- `Sass/SCSS`
- `Axios` 或 `ofetch`

### 选择理由

- Vue 官方站点将 Vue 描述为“approachable, performant and versatile framework”，适合构建可扩展的 Web UI。
- Vue 官方生态已明确包含 `Vue Router` 和 `Pinia`，适合标准化工程建设。
- Vite 官方文档说明其提供更快的开发体验、HMR 和生产构建能力，适合作为 Vue 3 工程基座。
- Pinia 轻量、模块化，适合商品、询盘、站点配置和页面状态管理。

## 4.2 后端

- `Node.js 24 LTS`
- `NestJS`
- `TypeScript`
- `Prisma ORM`
- `MySQL 8`
- `Redis`（可选）

### 选择理由

- 截至 2026-04-03，Node.js 官方发布页显示 `v24` 为 `Active LTS`，适合作为新项目生产环境基线。
- NestJS 官方文档明确说明其构建在 Express/Fastify 之上，强调模块化、可测试、可扩展和可维护，适合独立站后台与业务 API。
- Prisma 官方文档将其描述为支持类型安全、迁移和可视化数据编辑的 Node.js/TypeScript ORM，适合商品、分类、内容、询盘等关系型业务建模。
- MySQL 适合商品、分类、内容、询盘、订阅等标准业务数据结构。
- Redis 在本期不是必须，可用于缓存、限流、热点数据优化。

## 4.3 外围服务

- 文件存储：对象存储 + CDN
- 邮件：SMTP 或邮件服务商 API
- 统计：GA4、Meta Pixel

说明：

- 当前阶段不接入支付。
- 如后续扩展在线支付，可优先评估 Stripe 和 PayPal。

## 5. 总体架构

```text
用户浏览器
  -> Vue3 前端站点
  -> 调用 Node/NestJS API
  -> MySQL / Redis / 对象存储
  -> 第三方服务：邮件、统计

后台运营端
  -> Admin 管理系统
  -> Node/NestJS API
  -> MySQL / Redis / 对象存储
```

## 6. 前端架构设计

## 6.1 前端工程结构建议

```text
src/
  api/
  assets/
  components/
  composables/
  layouts/
  locales/
  pages/
  router/
  stores/
  styles/
  utils/
  App.vue
  main.ts
```

## 6.2 页面路由建议

- `/` 首页
- `/about` 品牌故事
- `/categories/:slug` 分类页
- `/products` 商品列表页
- `/products/:slug` 商品详情页
- `/search` 搜索页
- `/contact` 联系/批发合作
- `/policy/:type` 政策页

## 6.3 状态管理拆分

- `useAppStore`：站点配置、语言、货币、导航
- `useHomeStore`：首页模块数据
- `useProductStore`：商品列表、详情、筛选条件
- `useInquiryStore`：联系表单、批发表单、商品询盘
- `useCmsStore`：动态页面模块与 SEO 数据

## 6.4 样式实现策略

- 保留当前 CSS 变量体系，先抽离为全局设计 Token。
- 继续使用深蓝 + 橙色的品牌色体系。
- 大标题、按钮、卡片、圆角、阴影和背景渐变统一抽成样式规范。
- 不建议引入重型 UI 组件库，以免破坏当前品牌风格。
- 组件样式优先使用 `SCSS + BEM` 或 `Scoped SCSS`。

建议固化的 Token：

```css
:root {
  --navy: #10213a;
  --navy-2: #173053;
  --navy-3: #24456f;
  --orange: #ff6a2a;
  --orange-2: #ff8d47;
  --bg: #f4f7fb;
  --text: #10213a;
  --text-soft: #60708a;
}
```

## 6.5 国际化方案

- 默认语言英文，辅以中文。
- 文案放入 `locales/en.json` 与 `locales/zh-CN.json`。
- 前端首选根据用户手动切换语言，后续可扩展浏览器语言识别。
- 语言设置写入 `localStorage`，保持与现有原型行为一致。
- URL 方案可选：
  - MVP：统一 URL，前端切语言
  - SEO 增强版：`/en/...`、`/zh/...`

## 6.6 SEO 与渲染策略

独立站需要搜索流量，不建议纯 SPA 全站上线。

推荐策略：

- 首页、分类页、商品详情页采用 `SSR` 或 `预渲染`。
- 联系表单、订阅表单等交互页面使用 CSR。
- 每个页面支持动态 `title`、`description`、`canonical`、OG 信息。
- 商品详情页输出结构化数据。

如果项目首期预算有限，可采用：

- Vue3 + Vite 主工程
- 首页与主要营销页预渲染
- 商品详情页服务端生成 SEO 信息

## 6.7 动效方案

- Hero 自动轮播保留当前 5 秒级节奏。
- 滚动 reveal 动效继续使用 `IntersectionObserver`。
- 保留移动菜单展开、订阅浮层开合、按钮 hover 的轻量动效。
- 兼容 `prefers-reduced-motion`。

## 7. 后端架构设计

## 7.1 模块划分

建议按 NestJS 模块化拆分：

- `auth`：管理员登录、鉴权、权限
- `product`：商品、SKU、库存、标签
- `category`：分类树
- `cms`：首页模块、Banner、页面配置
- `inquiry`：联系/批发询盘
- `subscribe`：订阅邮箱
- `seo`：SEO 配置
- `system`：站点基础配置、字典、日志

## 7.2 API 风格

- 对外统一 RESTful API。
- 前后台接口逻辑分层，但共用统一服务层与数据模型。
- 接口版本化，建议前缀 `/api/v1`。

示例接口：

- `GET /api/v1/home`
- `GET /api/v1/categories`
- `GET /api/v1/products`
- `GET /api/v1/products/:slug`
- `POST /api/v1/inquiries`
- `POST /api/v1/product-inquiries`
- `POST /api/v1/subscriptions`
- `POST /api/v1/admin/auth/login`

## 7.3 数据库模型建议

核心数据表建议包括：

- `admin_users`
- `roles`
- `products`
- `product_skus`
- `product_media`
- `product_translations`
- `categories`
- `category_translations`
- `product_categories`
- `home_banners`
- `home_sections`
- `inquiries`
- `product_inquiries`
- `subscriptions`
- `seo_pages`
- `system_settings`
- `operation_logs`

## 7.4 内容管理策略

- 首页各模块均配置化，不写死在前端。
- 每个模块支持启用/停用、排序、标题、副标题、跳转链接、图片。
- 商品推荐位通过后台选择商品实现。
- 支持双语内容字段。

## 7.5 询盘与留资流程

```text
访问商品页或联系页
  -> 填写询盘/联系表单
  -> 服务端校验字段与反垃圾策略
  -> 写入 inquiry 或 product_inquiry
  -> 发送邮件通知
```

关键要求：

- 表单提交需校验必填项、邮箱格式和来源信息。
- 同一来源短时间重复提交需限流。
- 询盘状态、跟进记录和负责人分配需可追踪。

## 7.6 安全设计

- 后台采用 JWT + Refresh Token 或 Session 鉴权。
- 密码加密存储。
- 接口限流。
- 上传文件做类型和大小限制。
- 表单接口增加验证码或风控策略，防止垃圾提交。
- 使用参数化查询与 ORM，避免 SQL 注入。
- 管理后台记录关键操作日志。

## 8. 后台管理端设计

建议采用同一套 Vue 3 技术栈构建后台管理端，便于团队统一开发语言。

后台页面建议包括：

- 登录页
- 仪表盘
- 商品管理
- 分类管理
- 首页配置
- 询盘管理
- 订阅管理
- SEO 管理
- 系统配置

## 9. 性能优化方案

## 9.1 前端

- 图片压缩与 WebP/AVIF 优先输出。
- 首屏资源优先加载。
- 路由懒加载。
- 非关键组件异步加载。
- 使用 CDN 分发静态资源。

## 9.2 后端

- Redis 缓存首页配置、分类树、热点商品。
- 列表接口分页。
- 查询字段按页面裁剪，避免一次返回过多数据。
- 异步处理邮件、订阅同步和日志写入。

## 10. SEO 与营销技术点

- 生成 sitemap.xml
- 配置 robots.txt
- 配置 canonical
- 商品页输出结构化数据
- 支持 Open Graph
- 支持埋点事件上报
- 支持 UTM 参数透传与落库

## 14. 风险与建议

### 14.1 主要风险

- 如果前端做成纯 SPA，会削弱 SEO 效果。
- 如果首页模块数据继续写死，后期运营成本会很高。
- 如果直接引入通用 UI 框架，极易破坏当前已确定的品牌风格。
- 如果表单和询盘管理做得过轻，线索容易丢失或被垃圾提交污染。

### 14.2 明确建议

- 视觉上以当前静态稿为唯一基准。
- 技术上采用 Vue3 + Vite + Pinia + Vue I18n。
- 后端采用 Node.js + NestJS + Prisma + MySQL。
- SEO 重点页面必须支持 SSR 或预渲染。
- 当前阶段优先打通商品展示、询盘、订阅和后台内容配置四条主链路。
