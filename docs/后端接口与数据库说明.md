# MaxPlus 后端接口与数据库说明

## 1. 技术栈

- 后端框架: `Node.js + Express`
- 数据库: `MySQL 8+`
- 鉴权: `JWT`
- 密码存储: `bcrypt`
- 安全中间件: `helmet + cors + express-rate-limit`
- 敏感信息传输: `RSA + AES-GCM` 混合加密
- 敏感字段存储: 服务端 `AES-GCM` 二次加密

## 2. 目录说明

- 后端目录: `node/`
- 官网前端: `web/`
- 后台管理端: `sysadmin/`

## 3. 启动方式

### 3.1 后端

1. 复制 `node/.env.example` 为 `node/.env`
2. 配置 MySQL 连接信息
3. 安装依赖并启动

```bash
cd node
npm install
npm run dev
```

说明:

- 服务启动时会自动创建数据库、表结构和初始化种子数据
- 默认接口前缀为 `/api`
- 默认端口为 `4000`

### 3.2 官网前端

```bash
cd web
copy .env.example .env
npm install
npm run dev
```

### 3.3 后台管理端

```bash
cd sysadmin
copy .env.example .env
npm install
npm run dev
```

说明:

- 官网默认端口 `5173`
- 后台默认端口 `5174`

## 4. 安全设计

### 4.1 全接口统一使用 POST

后端当前所有业务接口均为 `POST`，满足项目约束。

### 4.2 敏感信息加密传输

以下敏感数据采用前端加密后再发送:

- 后台登录密码
- 官网询盘邮箱与留言
- 官网订阅邮箱与订单号
- 后台站点设置中的邮箱、电话等敏感配置

加密流程:

1. 前端调用 `POST /api/security/public-key` 获取 RSA 公钥
2. 前端随机生成 `AES-256-GCM` 会话密钥
3. 业务数据使用 AES 加密
4. AES 密钥再使用 RSA-OAEP 加密
5. 后端解密还原请求体

### 4.3 服务端安全措施

- `helmet` 安全响应头
- `express-rate-limit` 接口限流
- JWT 登录态校验
- MySQL 参数化查询，避免 SQL 注入
- 询盘与订阅敏感字段数据库内加密存储

## 6. 公共接口

### 6.1 获取前端加密公钥

- 地址: `POST /api/security/public-key`
- 认证: 否
- 请求体:

```json
{}
```

- 响应体:

```json
{
  "success": true,
  "message": "OK",
  "data": {
    "publicKeyPem": "-----BEGIN PUBLIC KEY-----...",
    "keyId": "xxxxxxxxxxxxxxxx"
  }
}
```

### 6.2 官网内容引导数据

- 地址: `POST /api/public/bootstrap`
- 认证: 否
- 说明: 官网首页、产品、分类、视频、博客等页面的数据入口

### 6.3 前台询盘提交

- 地址: `POST /api/public/inquiry`
- 认证: 否
- 传输: 建议前端使用加密 envelope
- 请求体字段:

```json
{
  "source": "product",
  "name": "Ryan",
  "email": "ryan@example.com",
  "company": "Fieldhouse Supply",
  "message": "Need quote for 50 sets.",
  "sourceDetail": "Portable Volleyball Net Kit"
}
```

### 6.4 前台订阅提交

- 地址: `POST /api/public/subscribe`
- 认证: 否
- 传输: 建议前端使用加密 envelope
- 请求体字段:

```json
{
  "email": "user@example.com",
  "orderNumber": "MP-10086",
  "source": "Website Subscribe Widget"
}
```

## 7. 登录接口

### 7.1 后台登录

- 地址: `POST /api/auth/login`
- 认证: 否
- 传输: 必须加密
- 请求体:

```json
{
  "username": "admin",
  "password": "maxplus123",
  "role": "super-admin"
}
```

- 响应体:

```json
{
  "success": true,
  "message": "登录成功。",
  "data": {
    "token": "jwt-token",
    "session": {
      "id": "usr-admin",
      "username": "admin",
      "role": "super-admin",
      "loggedInAt": "2026-04-03T12:00:00.000Z"
    }
  }
}
```

## 8. 后台管理接口

以下接口都需要:

- 请求方法: `POST`
- 请求头: `Authorization: Bearer <token>`

### 8.1 后台初始化数据

- 地址: `POST /api/admin/bootstrap`
- 说明: 后台登录后一次性拉取所有管理数据

### 8.2 仪表盘

- `POST /api/admin/dashboard`

### 8.3 站点设置

- 获取: `POST /api/admin/site/get`
- 保存: `POST /api/admin/site/save`

请求体结构:

```json
{
  "brand": {
    "brandName": "MaxPlus Sporting Goods",
    "siteTitle": "MaxPlus Sporting Goods",
    "siteDescription": "Sporting goods brand site...",
    "supportEmail": "support@maxplus-sport.com",
    "salesEmail": "hello@maxplus-sport.com",
    "phone": "+1 (800) 555-0199",
    "whatsapp": "+1 800 555 0199",
    "defaultLanguage": "English",
    "defaultCurrency": "USD",
    "timezone": "America/Los_Angeles",
    "address": "Los Angeles, California, United States"
  },
  "socials": [
    {
      "name": "Instagram",
      "url": "https://instagram.com/maxplus"
    }
  ],
  "notifications": {
    "inquiryRecipients": "sales@maxplus-sport.com",
    "subscriberRecipients": "crm@maxplus-sport.com",
    "enableEmailNotice": true,
    "enableSlackNotice": false
  }
}
```

### 8.4 首页内容

- 获取: `POST /api/admin/home/get`
- 保存: `POST /api/admin/home/save`

### 8.5 SEO 设置

- 获取: `POST /api/admin/seo/get`
- 保存: `POST /api/admin/seo/save`

### 8.6 商品/分类/视频/博客

- 获取全部内容: `POST /api/admin/catalog/get`

分类:

- 保存: `POST /api/admin/categories/save`
- 删除: `POST /api/admin/categories/delete`

商品:

- 保存: `POST /api/admin/products/save`
- 删除: `POST /api/admin/products/delete`

视频:

- 保存: `POST /api/admin/videos/save`
- 删除: `POST /api/admin/videos/delete`

博客:

- 保存: `POST /api/admin/blogs/save`
- 删除: `POST /api/admin/blogs/delete`

### 8.7 CRM

询盘:

- 列表: `POST /api/admin/inquiries/list`
- 更新状态/负责人: `POST /api/admin/inquiries/update`

请求体:

```json
{
  "id": "inq-1",
  "status": "processing",
  "assignee": "Derek"
}
```

订阅:

- 列表: `POST /api/admin/subscribers/list`

### 8.8 日志

- 列表: `POST /api/admin/logs/list`

## 9. 统一响应格式

成功:

```json
{
  "success": true,
  "message": "OK",
  "data": {}
}
```

失败:

```json
{
  "success": false,
  "message": "错误信息",
  "details": null
}
```

## 10. 数据库表设计

核心表:

- `admin_users`: 后台用户
- `site_settings`: 站点设置
- `home_content`: 首页内容配置
- `seo_settings`: SEO 设置
- `categories`: 分类
- `products`: 商品
- `videos`: 视频
- `blogs`: 博客
- `inquiries`: 询盘
- `subscribers`: 订阅
- `logs`: 系统日志

说明:

- 商品、分类、博客、视频中的复杂字段使用 `JSON` 列存储
- 询盘和订阅中的敏感字段使用服务端 AES 加密后落库
